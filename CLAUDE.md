# 프로젝트

수치해석 수업 실습·과제 저장소. 방정식의 근을 찾는 고전 수치해법을 Python으로 직접 구현했다.

## 환경·실행

- 언어: Python 3
- 주요 패키지: `math`, `numpy`, `matplotlib`, 일부 `scipy` (`scipy.misc.derivative` 등)
- 실행(.py): `python "algorithms/Bisection Algorithm.py"` 처럼 파일 단위 실행. 대부분 함수 `f(x)`·초기값·허용오차(TOL)를 스크립트 상단에 하드코딩
- 실습(.ipynb): Jupyter/VS Code 셀 실행
- 별도 빌드/테스트 없음

## 폴더 구조

- `algorithms/` — 근 찾기 알고리즘 구현 `.py`
  - 이분법, 가위치법(False position), 고정점 반복, 뉴턴법(중근 처리 포함), 할선법(Secant)
- `exercises/` — 교재 장별 연습문제·과제 노트북 (`2_*`, `3_*`, `수치해석.ipynb`)

## 컨벤션

- 알고리즘 파일은 스크립트 최상단에서 대상 함수 `f(x)`, 구간 `[a,b]`/초기값, `TOL`(예: `10**-4`)을 정의하고 while 루프로 반복
- 반복 과정 값을 리스트(`p1`, `y1` 등)에 누적해 수렴을 관찰하는 스타일 유지
- 파일명은 알고리즘명(영문) 또는 교재 문제 번호(`exercise 3_3_2.ipynb`, `2_장 과제.ipynb`) 기준. 한글·공백 포함 가능

## 주의

- 각 알고리즘은 특정 예제 함수에 맞춰 초기값·구간이 하드코딩돼 있다. 다른 문제에 재사용하려면 `f`·범위·TOL을 함께 바꿔야 하며, 잘못된 초기 구간은 발산/무한루프를 유발할 수 있다
- 뉴턴법·할선법은 도함수/이전 점 설정이 함수마다 수기로 작성돼 있으니 `f`만 바꾸면 안 됨
- 한글 파일명은 NFC로 통일할 것
- `.ipynb_checkpoints/`, `__pycache__/`, `.DS_Store` 커밋 금지
